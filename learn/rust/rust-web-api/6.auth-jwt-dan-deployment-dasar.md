# Auth JWT dan deployment dasar

## Alur JWT singkat

1. User login dengan email/password.
2. Server memverifikasi user.
3. Server menghasilkan access token JWT.
4. Endpoint private memverifikasi token di header Authorization.

## Dependency contoh

```toml
jsonwebtoken = "9"
chrono = "0.4"
```

## Deployment dasar

- Build release:

```bash
cargo build --release
```

- Simpan env penting di server:

`DATABASE_URL`, `JWT_SECRET`, `RUST_LOG`

- Jalankan binary:

```bash
./target/release/rust_api
```

## Checklist produksi minimum

- Gunakan reverse proxy (Nginx/Caddy).
- Aktifkan HTTPS.
- Tambahkan logging request dan error.
- Pasang health endpoint (`/health`) dan monitoring.
