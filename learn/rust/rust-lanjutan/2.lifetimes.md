# Lifetimes

Lifetime memastikan reference valid selama masa pakainya.
Biasanya compiler bisa infer otomatis, tapi di beberapa fungsi kita perlu menuliskannya.

## Contoh basic lifetime annotation

```rust
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
```

## Kenapa penting

- Mencegah dangling reference.
- Membuat hubungan usia antar reference jadi eksplisit.

## Tips belajar

- Mulai dari kasus sederhana dua parameter reference.
- Baca error compiler perlahan karena pesan lifetime Rust sangat membantu.
