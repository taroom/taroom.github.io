Web Worker

HTML5 memperkenalkan konsep Web Worker, yang memungkinkan kita untuk menjalankan skrip di background thread terpisah. Ini sangat berguna untuk melakukan tugas berat tanpa mengganggu UI utama. Dalam tutorial ini, kita akan membahas cara menggunakan Web Worker, jenis-jenisnya, dan beberapa best practices.

Multithreading di JavaScript

Web Workers memungkinkan eksekusi script di background thread terpisah tanpa memblokir UI thread utama.

Konsep Dasar
Karakteristik Web Workers:
  • Thread Terpisah - Berjalan paralel dengan main thread
  • Tidak Blocking UI - Operasi berat tidak freeze browser
  • Komunikasi via Messaging - postMessage dan onmessage
  • No DOM Access - Tidak bisa mengakses DOM/window
  • Dedicated/Shared - Worker khusus atau bisa dibagi
Kapan Menggunakan:
  • Proses berat (kalkulasi kompleks)
  • Operasi I/O besar (parsing file)
  • Pre-fetching data
  • Image/video processing
  • Machine learning di client
Batasan:
  • Tidak bisa mengakses DOM
  • Terbatas akses ke window/document
  • Same-origin policy
  • File worker harus dari sumber yang sama
Implementasi Dasar
File Utama (main.js):
// Membuat worker
          const worker = new Worker('worker.js');
          
          // Mengirim data ke worker
          worker.postMessage({ 
            command: 'calculate', 
            data: 1000000 
          });
          
          // Menerima hasil dari worker
          worker.onmessage = function(e) {
            console.log('Hasil:', e.data.result);
            
            // Menghentikan worker
            worker.terminate();
          };
          
          // Menangani error
          worker.onerror = function(e) {
            console.error('Error:', e.message);
          };
File Worker (worker.js):
// Menerima pesan dari main thread
          onmessage = function(e) {
            if (e.data.command === 'calculate') {
              // Lakukan kalkulasi berat
              const result = heavyCalculation(e.data.data);
              
              // Kirim kembali hasil
              postMessage({ 
                result: result,
                status: 'completed' 
              });
            }
          };
          
          function heavyCalculation(n) {
            let sum = 0;
            for (let i = 0; i < n; i++) {
              sum += Math.sqrt(i) * Math.random();
            }
            return sum;
          }

Jenis-Jenis Workers
Dedicated Worker

Worker khusus untuk satu window/tab:

// Main script
          const worker = new Worker('worker.js');
  • Komunikasi 1-1
  • Hidup selama parent
  • Lebih sederhana
Shared Worker

Bisa dibagi antar window/tab:

// Main script
          const worker = new SharedWorker('shared.js');
  • Komunikasi via port
  • Bisa multi-koneksi
  • Lebih kompleks
Service Worker

Untuk offline caching & push notif:

// Main script
          navigator.serviceWorker.register('sw.js');
  • Proxy jaringan
  • Event-driven
  • PWA requirement

Best Practices:

  • Gunakan untuk tugas berat yang benar-benar membutuhkan
  • Minimalkan komunikasi antara worker dan main thread
  • Transfer data besar dengan Transferable Objects
  • Hindari membuat terlalu banyak worker (browser limits)
  • Gunakan error handling yang baik
  • Terminate worker ketika tidak lagi dibutuhkan
Transferable Objects
Konsep Transferable:

Memindahkan kepemilikan data antara thread tanpa copy:

  • ArrayBuffer
  • MessagePort
  • ImageBitmap
  • OffscreenCanvas

Setelah ditransfer, data tidak bisa diakses lagi di thread asal.

Contoh Implementasi:
// Main thread
          const buffer = new ArrayBuffer(1024 * 1024 * 32); // 32MB
          worker.postMessage(
            { buffer: buffer },
            [buffer] // Transfer ownership
          );
          
          // Worker
          onmessage = function(e) {
            const buffer = e.data.buffer; // Sekarang dimiliki worker
            // Proses buffer...
            
            // Kembalikan hasil
            postMessage(
              { result: processedBuffer },
              [processedBuffer] // Transfer kembali
            );
          };
Sangat efisien untuk data besar seperti gambar, audio, atau video.
Library & Tools Pendukung
Comlink

Membuat worker seolah-objek lokal:

// Worker
          const obj = { add: (a, b) => a + b };
          expose(obj);
          
          // Main thread
          const obj = await wrap(worker);
          console.log(await obj.add(1, 2)); // 3
Workerize

Membuat worker dari modul ES6:

// worker.js
          export function add(a, b) { return a + b; }
          
          // main.js
          const worker = workerize('./worker.js');
          const result = await worker.add(1, 2);
Threads.js

API thread-like untuk browser & Node:

const { spawn, Thread } = require('threads');
          const worker = await spawn(new Worker('./worker'));
          const result = await worker.doWork();
