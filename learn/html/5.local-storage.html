Web Storage API

HTML5 memperkenalkan API penyimpanan lokal yang memungkinkan aplikasi web menyimpan data di browser pengguna.

Penyimpanan Client-Side

HTML5 memperkenalkan Web Storage API sebagai pengganti cookie yang lebih powerful untuk menyimpan data di browser pengguna.

Jenis Web Storage
Fitur localStorage sessionStorage Cookies
Kapasitas ~5-10MB ~5-10MB ~4KB
Durasi Permanen (sampai dihapus) Sesi browser/tab Bisa diset expiry
Akses Server Tidak Tidak Ya (setiap request)
Syntax setItem(), getItem() document.cookie
Kapan Menggunakan:
  • localStorage: Penyimpanan preferensi user, data aplikasi offline
  • sessionStorage: Data sementara selama sesi browsing
  • Cookies: Autentikasi dan data kecil yang perlu diakses server
localStorage
Operasi Dasar:

// Menyimpan data
localStorage.setItem('nama', 'John Doe');
localStorage.setItem('umur', 30);

// Membaca data
const nama = localStorage.getItem('nama');
const umur = localStorage.getItem('umur');

// Menghapus data
localStorage.removeItem('nama');

// Menghapus semua
localStorage.clear();
Demo Interaktif:
Penyimpanan Objek:

// Menyimpan objek
const user = { nama: 'John', id: 123 };
localStorage.setItem('user', JSON.stringify(user));

// Membaca objek
const savedUser = JSON.parse(localStorage.getItem('user'));
localStorage hanya menyimpan string. Gunakan JSON.stringify() dan JSON.parse() untuk objek.
sessionStorage
Karakteristik:
  • Data bertahan selama sesi browser/tab
  • Data hilang saat tab ditutup
  • Syntax sama dengan localStorage
  • Cocok untuk data sementara
Contoh Penggunaan:

// Menyimpan data sementara
sessionStorage.setItem('token', 'abc123xyz');

// Mendapatkan data
const token = sessionStorage.getItem('token');

IndexedDB
Konsep Dasar:
  • Database NoSQL client-side
  • Kapasitas besar (~50MB+ tergantung browser)
  • Mendukung transaksi dan indexing
  • API lebih kompleks tapi lebih powerful
Kapan Menggunakan:
  • Aplikasi offline kompleks
  • Penyimpanan data terstruktur
  • Ketika butuh query kompleks
  • Untuk data besar di client
Contoh Kode:

                            // Membuka database
const request = indexedDB.open('MyDatabase', 1);

request.onupgradeneeded = (event) => {
  const db = event.target.result;
  const store = db.createObjectStore('users', { keyPath: 'id' });
  store.createIndex('nama', 'nama', { unique: false });
};

request.onsuccess = (event) => {
  const db = event.target.result;
  const tx = db.transaction('users', 'readwrite');
  const store = tx.objectStore('users');
  
  // Menambahkan data
  store.add({ id: 1, nama: 'John', email: 'john@example.com' });
};
Gunakan library seperti Dexie.js untuk menyederhanakan IndexedDB.

Best Practices:

  • Gunakan try-catch karena storage bisa penuh atau diblokir
  • Hindari menyimpan data sensitif (password, dll)
  • Bersihkan data yang tidak perlu secara berkala
  • Untuk data kompleks, pertimbangkan IndexedDB
  • Selalu beri prefix pada key (misal: 'myapp_settings')
