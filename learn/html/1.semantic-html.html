Struktur Semantik HTML5

Memahami struktur semantik HTML5 untuk pengembangan web yang lebih baik.

Apa itu Elemen Semantik?

Elemen semantik adalah elemen HTML yang secara jelas mendeskripsikan maknanya baik untuk browser maupun pengembang.

Header & Footer 
<header>
        <!-- Konten kepala halaman -->
      </header>
      
      <footer>
        <!-- Konten kaki halaman -->
      </footer>

Digunakan untuk mengelompokkan konten kepala dan kaki halaman.

Navigasi 
<nav>
        <ul>
          <li><a href="#">Home</a></li>
          <li><a href="#">About</a></li>
        </ul>
      </nav>

Khusus untuk menu navigasi utama.

Article & Section 
<article>
        <h2>Judul Artikel</h2>
        <p>Konten artikel...</p>
      </article>
      
      <section>
        <h2>Bagian 1</h2>
        <p>Konten bagian...</p>
      </section>

<article> untuk konten mandiri, <section> untuk pengelompokan tema.

Aside & Main 
<main>
        <!-- Konten utama -->
      </main>
      
      <aside>
        <!-- Konten samping/sidebar -->
      </aside>

<main> untuk konten utama, <aside> untuk konten terkait.

Manfaat Struktur Semantik:

  • SEO Lebih Baik - Mesin pencari lebih mudah memahami struktur konten
  • Aksesibilitas - Screen reader dapat menavigasi halaman dengan lebih baik
  • Kode Lebih Terorganisir - Memudahkan pengembangan dan maintenance
  • Responsif - Mempermudah styling untuk berbagai perangkat

Contoh Layout HTML5 Semantik Lengkap: 

<!DOCTYPE html>
      <html lang="id">
      <head>
          <meta charset="UTF-8">
          <title>Contoh Layout Semantik</title>
      </head>
      <body>
          <header>
              <h1>Judul Situs</h1>
              <nav>...</nav>
          </header>
          
          <main>
              <article>
                  <h2>Judul Artikel</h2>
                  <section>
                      <h3>Bagian 1</h3>
                      <p>Konten...</p>
                  </section>
              </article>
              
              <aside>
                  <h3>Sidebar</h3>
                  <p>Informasi terkait...</p>
              </aside>
          </main>
          
          <footer>
              <p>Hak Cipta © 2023</p>
          </footer>
      </body>
      </html>

Best Practices:

  • Gunakan <main> hanya sekali per halaman
  • Hindari nesting <article> di dalam <article>
  • <section> harus selalu memiliki heading
  • Kombinasikan dengan ARIA roles untuk aksesibilitas maksimal
