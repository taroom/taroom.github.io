Web Socket

Web Socket adalah protokol komunikasi yang menyediakan saluran komunikasi full-duplex melalui koneksi TCP. Ini memungkinkan pertukaran data secara real-time antara klien dan server.

Web Socket digunakan dalam aplikasi web yang memerlukan komunikasi real-time, seperti aplikasi chat, game online, dan aplikasi kolaborasi.

Komunikasi Real-Time Dua Arah

WebSocket menyediakan koneksi persisten dua arah antara client dan server, ideal untuk aplikasi real-time.

Dasar WebSocket
Perbandingan dengan HTTP:
Aspek HTTP WebSocket
Koneksi Stateless, terputus setelah request Persisten, terus-menerus
Komunikasi Client → Server saja Dua arah (full-duplex)
Overhead Header lengkap setiap request Header minimal setelah handshake
Use Case Halaman web tradisional Aplikasi real-time
Lifecycle WebSocket:
  1. Handshake - Koneksi awal via HTTP upgrade
  2. Connected - Koneksi terbuka (onopen)
  3. Communication - Pertukaran data (onmessage)
  4. Closing - Penutupan koneksi (onclose)
  5. Error Handling - Tangani error (onerror)
URI Scheme:
ws:// (unencrypted) atau wss:// (encrypted, rekomendasi)
Implementasi Client
Koneksi Dasar:

// Buat koneksi WebSocket
const socket = new WebSocket('wss://example.com/socket');

// Event saat koneksi terbuka
socket.addEventListener('open', (event) => {
    console.log('Connected to server');
    socket.send('Hello Server!');
});

// Event menerima pesan
socket.addEventListener('message', (event) => {
    console.log('Message from server:', event.data);
});

// Event saat koneksi tertutup
socket.addEventListener('close', (event) => {
    console.log('Disconnected:', event.code, event.reason);
});

// Event error
socket.addEventListener('error', (error) => {
    console.error('WebSocket error:', error);
});
Method Utama:
  • send(data) - Mengirim data ke server
  • close(code?, reason?) - Menutup koneksi
Status Koneksi:
  • socket.readyState:
    • 0: CONNECTING
    • 1: OPEN
    • 2: CLOSING
    • 3: CLOSED

Implementasi Server
Node.js dengan ws Library:

const WebSocket = require('ws');
const wss = new WebSocket.Server({ port: 8080 });

wss.on('connection', (ws) => {
    console.log('New client connected');

    // Terima pesan dari client
    ws.on('message', (message) => {
        console.log('Received:', message);
        
        // Broadcast ke semua client
        wss.clients.forEach((client) => {
            if (client.readyState === WebSocket.OPEN) {
                client.send(message);
            }
        });
    });

    // Kirim welcome message
    ws.send('Welcome to WebSocket server!');
});
PHP dengan Ratchet:
use Ratchet\MessageComponentInterface;
              use Ratchet\ConnectionInterface;
              
              class Chat implements MessageComponentInterface {
                  protected $clients;
              
                  public function __construct() {
                      $this->clients = new \SplObjectStorage;
                  }
              
                  public function onOpen(ConnectionInterface $conn) {
                      $this->clients->attach($conn);
                  }
              
                  public function onMessage(ConnectionInterface $from, $msg) {
                      foreach ($this->clients as $client) {
                          $client->send($msg);
                      }
                  }
              
                  public function onClose(ConnectionInterface $conn) {
                      $this->clients->detach($conn);
                  }
              
                  public function onError(ConnectionInterface $conn, \Exception $e) {
                      $conn->close();
                  }
              }
              
              // Run server
              $app = new Ratchet\App('localhost', 8080);
              $app->route('/chat', new Chat);
              $app->run();

Best Practices:

  • Gunakan wss:// untuk koneksi aman (HTTPS)
  • Implementasi reconnect logic untuk koneksi yang terputus
  • Gunakan heartbeat/ping-pong untuk memantau koneksi
  • Format data sebagai JSON untuk pesan kompleks
  • Batas ukuran pesan untuk menghindari DoS
  • Gunakan compression untuk data besar
Alternatif WebSocket
Server-Sent Events (SSE)

Komunikasi satu arah dari server ke client:


const eventSource = new EventSource('/updates');
eventSource.onmessage = (e) => {
    console.log(e.data);
};

Cocok untuk notifikasi real-time sederhana.

Long Polling

HTTP polling dengan koneksi tetap terbuka:


function poll() {
    fetch('/updates')
    .then(response => response.json())
    .then(data => {
        process(data);
        poll(); // Polling lagi
    });
}

Fallback ketika WebSocket tidak tersedia.
