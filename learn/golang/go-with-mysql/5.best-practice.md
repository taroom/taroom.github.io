# Best practice query dan error handling

## Praktik yang direkomendasikan

- Selalu gunakan context untuk query database.
- Batasi timeout query agar aplikasi tidak menggantung.
- Gunakan transaksi untuk proses multi-query yang saling terkait.
- Pisahkan error internal dan pesan error untuk client.

## Contoh query dengan timeout

```go
ctx, cancel := context.WithTimeout(context.Background(), 3*time.Second)
defer cancel()

row := db.QueryRowContext(ctx, "SELECT id, name, email FROM users WHERE id = ?", id)
```

## Contoh transaksi

```go
tx, err := db.Begin()
if err != nil {
    return err
}

if _, err := tx.Exec("UPDATE accounts SET balance = balance - ? WHERE id = ?", amount, fromID); err != nil {
    _ = tx.Rollback()
    return err
}

if _, err := tx.Exec("UPDATE accounts SET balance = balance + ? WHERE id = ?", amount, toID); err != nil {
    _ = tx.Rollback()
    return err
}

return tx.Commit()
```
